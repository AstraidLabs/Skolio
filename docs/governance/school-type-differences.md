# Skolio — School-Type Differences (Phase 8)

## Kindergarten
- Preferovaný provozní model: skupiny, attendance, daily reports, komunikace s rodiči.
- UI priorita: group-centric přehledy a denní provozní workflow.
- Omezení: nepřebírá secondary-school model předmětového plánování jako primární osu.

## ElementarySchool
- Preferovaný provozní model: třídy, předměty, rozvrh, attendance, grades, homework.
- UI priorita: class/subject flows s konzistentními list/detail/create/edit obrazovkami.
- Omezení: bez semestrů, kreditů, exam workflows.

## SecondarySchool
- Preferovaný provozní model: elementary schopnosti + obory a ročníky.
- UI priorita: branch/year organizace bez univerzitních konceptů.
- Omezení: žádný enrollment/credits/semester engine.

## Společné pravidlo
- Není tvořena jedna univerzální abstrakce, která by potlačila rozdíly.
- Současně nevznikají tři oddělené aplikace; sdílí se platformní technický základ a role model.
